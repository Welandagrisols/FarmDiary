# PHASE 1 — The Working Farm Diary
## Potato Farm Management App | Replit + Supabase + React PWA
### Build this first. It is complete and usable on its own.

---

## WHAT THIS PHASE BUILDS

A mobile-first app that replaces the farm manager's notebook. It shows the spray and activity schedule, lets you log what was actually done, captures every cost (including historical costs from before the app existed), and tracks inventory. By the end of this phase you have a fully working tool for the current season.

**What is NOT in this phase** (added later): multiple farms, multiple seasons, satellite maps, GPS walk, full P&L reports, Excel import. The database is designed so all of these slot in cleanly in later phases without restructuring.

---

## TECH STACK

- **Frontend**: React with Vite + Tailwind CSS
- **Database**: Supabase (PostgreSQL)
- **Auth**: None yet — single user, no login. Add `user_id UUID` as a nullable column on every table so auth can be bolted on later with zero migration.
- **Hosting**: Replit
- **PWA**: Add `manifest.json` + basic service worker so the app is installable on Android
- **Charts**: Recharts
- **Icons**: Lucide React

## DATABASE SCHEMA (Phase 1)

Design these tables to support multi-farm and multi-season in future phases. Include the foreign key columns now even though Phase 1 only uses one farm and one season.

```sql
-- Farms table (Phase 1 seeds one farm, Phase 2 adds the UI to manage more)
CREATE TABLE farms (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  name TEXT NOT NULL DEFAULT 'Rift Valley Potato Farm',
  location TEXT DEFAULT 'Nakuru / Bomet / Kericho, Kenya',
  total_acres NUMERIC DEFAULT 4,
  lease_status TEXT DEFAULT 'Leased',
  -- Future phase columns (include now, leave null):
  latitude NUMERIC,
  longitude NUMERIC,
  boundary_geojson JSONB,
  area_acres_calculated NUMERIC,
  notes TEXT,
  user_id UUID,
  created_at TIMESTAMP DEFAULT NOW()
);

-- Seasons table (Phase 1 seeds one season)
CREATE TABLE seasons (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  farm_id UUID REFERENCES farms(id) ON DELETE CASCADE,
  season_name TEXT NOT NULL,
  season_type TEXT,           -- 'Long Rains', 'Short Rains', 'Dry Season'
  status TEXT DEFAULT 'Active',
  notes TEXT,
  user_id UUID,
  created_at TIMESTAMP DEFAULT NOW()
);

-- Sections (the subdivisions within a season — Section A and Section B)
CREATE TABLE sections (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  season_id UUID REFERENCES seasons(id) ON DELETE CASCADE,
  farm_id UUID REFERENCES farms(id),
  label TEXT NOT NULL,
  variety TEXT,
  acres NUMERIC,
  planting_date DATE,
  estimated_germination DATE,
  actual_germination DATE,
  estimated_harvest DATE,
  actual_harvest DATE,
  blight_risk TEXT DEFAULT 'MEDIUM',
  notes TEXT,
  -- Future phase:
  boundary_geojson JSONB,
  user_id UUID,
  created_at TIMESTAMP DEFAULT NOW()
);

-- The master cost ledger — every single expenditure goes here
CREATE TABLE costs (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  farm_id UUID REFERENCES farms(id),
  season_id UUID REFERENCES seasons(id),
  section_id UUID REFERENCES sections(id),  -- NULL = whole farm level

  -- What is this cost?
  cost_category TEXT NOT NULL,
  cost_subcategory TEXT,
  description TEXT NOT NULL,

  -- When did it happen?
  cost_date DATE NOT NULL,
  is_pre_planting BOOLEAN DEFAULT FALSE,
  is_historical BOOLEAN DEFAULT FALSE,      -- TRUE if entered retrospectively

  -- How much?
  amount_kes NUMERIC NOT NULL,
  quantity NUMERIC,
  unit TEXT,
  unit_price_kes NUMERIC,

  -- Product detail (for chemicals and inputs)
  product_name TEXT,
  supplier TEXT,
  receipt_reference TEXT,

  -- Labor detail
  num_workers INT,
  days_worked NUMERIC,
  rate_per_worker_per_day NUMERIC,

  -- Facilitation detail
  facilitator_name TEXT,
  trip_from TEXT,
  trip_to TEXT,

  -- Deviation from plan?
  is_deviation BOOLEAN DEFAULT FALSE,
  planned_product TEXT,
  deviation_reason TEXT,

  -- Import tracking
  import_source TEXT DEFAULT 'manual',

  notes TEXT,
  photo_url TEXT,
  weather_conditions TEXT,
  user_id UUID,
  created_at TIMESTAMP DEFAULT NOW()
);

-- Inventory: what was bought and how much is left
CREATE TABLE inventory (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  farm_id UUID REFERENCES farms(id),
  season_id UUID REFERENCES seasons(id),
  product_name TEXT NOT NULL,
  category TEXT,
  quantity_purchased NUMERIC NOT NULL,
  unit TEXT NOT NULL,
  unit_price_kes NUMERIC NOT NULL,
  quantity_used NUMERIC DEFAULT 0,
  purchase_date DATE NOT NULL,
  supplier TEXT,
  low_stock_threshold NUMERIC,
  is_historical BOOLEAN DEFAULT FALSE,
  notes TEXT,
  user_id UUID,
  created_at TIMESTAMP DEFAULT NOW()
);

-- Activity logs: when was a scheduled activity actually done?
CREATE TABLE activity_logs (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  farm_id UUID REFERENCES farms(id),
  season_id UUID REFERENCES seasons(id),
  section_id UUID REFERENCES sections(id),
  schedule_activity_id TEXT,          -- links to hardcoded PLANNED_SCHEDULE id
  activity_name TEXT NOT NULL,
  planned_date DATE,
  actual_date DATE NOT NULL,
  products_used JSONB,
  is_deviation BOOLEAN DEFAULT FALSE,
  deviation_reason TEXT,
  num_workers INT,
  labor_cost_kes NUMERIC DEFAULT 0,
  total_cost_kes NUMERIC DEFAULT 0,
  weather_conditions TEXT,
  is_historical BOOLEAN DEFAULT FALSE,
  notes TEXT,
  user_id UUID,
  created_at TIMESTAMP DEFAULT NOW()
);

-- Field observations: daily scouting notes
CREATE TABLE field_observations (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  farm_id UUID REFERENCES farms(id),
  season_id UUID REFERENCES seasons(id),
  section_id UUID REFERENCES sections(id),
  observation_date DATE NOT NULL,
  observation_type TEXT,
  description TEXT NOT NULL,
  severity TEXT,
  action_taken TEXT,
  is_historical BOOLEAN DEFAULT FALSE,
  user_id UUID,
  created_at TIMESTAMP DEFAULT NOW()
);
```

**Seed data**: Insert one farm, one season (Long Rains 2026), and two sections (A and B) on app first load if tables are empty.

## SCREENS TO BUILD (Phase 1)

### Navigation
Bottom bar with 5 tabs: **Home | Schedule | Log | Costs | More**
"More" opens: Inventory | Observations | Settings (placeholder)

---

### Screen 1: Dashboard (Home)

- **Section cards** — one for Section A (Stephen's) and one for Section B (Shangi):
  - Variety + acres
  - Days since planting
  - Current growth stage (calculated from planting date)
  - Next planned activity: name + planned date + days until due
  - Urgency color: 🟢 >7 days | 🟡 3-7 days | 🔴 ≤3 days or overdue
- **Budget bar**: "KES X spent of KES 83,850 estimated" with a progress bar
- **Alert banner** (red, dismissible): shows if any activity is >3 days overdue OR if Stephen's hasn't been sprayed in >7 days
- **Two quick action buttons**: [+ Log Activity] [+ Add Cost]

---

### Screen 2: Schedule

- Chronological list of all planned activities for the current season
- Each card shows: Stage badge | Name | Sec A date | Sec B date | Status chip
- Status chips: Completed ✅ | Overdue 🔴 | Due in ≤3 days 🟡 | Upcoming ⬜
- Tap a card → detail modal: purpose text, products list with rates, alert message
- "Log Actual →" button on each card → pre-fills the Log screen for that activity

---

### Screen 3: Log Activity

**Step-by-step form — make it fast for field use.**

Step 1: What are you logging?
- Choose activity: dropdown from PLANNED_SCHEDULE (shows name + due date) OR "Other / Unplanned" OR "Historical Entry (past date)"
- Section: [Sec A] [Sec B] [Both]

Step 2: Date & Weather
- Date picker (defaults to today; allows any past date)
- Weather: ☀️ Sunny | 🌦️ Light Rain | 🌧️ Heavy Rain | ☁️ Cloudy

Step 3: Products Used
- Pre-filled from the selected planned activity
- Each product row: toggle [✅ Used as planned] or [⚠️ Used different product]
  - If different: text field for actual product + reason dropdown:
    "Product unavailable | Price too high — substituted | Heavy rain — used rain-fast alternative | Agronomist advice | Other"
- Qty used | Unit | Price per unit → auto-calculates line total
- [+ Add another product] button for ad-hoc additions

Step 4: Labor
- Number of workers (number input)
- Daily rate per worker (KES) — default 500
- Days worked (default 1)
- Auto-calculates: workers × rate × days = total labor KES

Step 5: Other Costs (collapsible section)
- Each row: Type (Transport / Meals / Accommodation / Equipment / Other) | Description | Amount KES
- [+ Add cost row] button

Step 6: Notes
- Free text notes field

[SUBMIT — large green full-width button]

On submit: create one `activity_logs` row + one or more `costs` rows (products, labor, other costs each become separate cost rows linked to the same activity log). Update inventory `quantity_used` for each product logged.

---

### Screen 4: Costs (The Ledger)

**Sub-tabs**: All | Pre-Planting | Inputs | Labor | Facilitation | Logistics | Equipment

Top summary row:

## BUSINESS RULES

```
Section A (Stephen's) planted: 17 February 2026
Section B (Shangi) planted: 20 February 2026
Section A germination est.: 8 March 2026
Section B germination est.: 11 March 2026

SPRAY INTERVALS:
- Stage 1: 12 days standard | 7 days if heavy daily rain
- Stage 2: 12 days standard | 7 days if heavy daily rain
- Stage 3: 14 days standard | 7 days if heavy daily rain
- Stephen's: NEVER exceed 12 days without a spray
- Alert if any spray is overdue by more than 3 days

PRODUCT RULES:
- Kenthane: Stage 1 ONLY. Never in Stage 2 or 3.
- Mastergold + Metameta: Always blended. Stage 2 onward.
- Carbosink ≠ Carbosink Calcium (two different products)
- Halothrin + Vendex: Always logged as a combined blend
- Rabbit Urine: Max 20L stock. Warn if exceeded in inventory.
- Sands and Surds: Same product. Use "Sands (Sticker/Spreader)"

COST RULES:
- All costs in KES only
- Any cost with a past date → is_historical = TRUE automatically
- is_pre_planting = TRUE if cost_date < planting_date of section
- user_id column exists on all tables but is always NULL in Phase 1
```
